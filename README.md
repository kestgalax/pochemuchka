# Почемучка

> Найди корень. Убери посредника. Окажись там, где проблемы нет.

**Почемучка** — это метод и навык для тех, кто хочет не «решить проблему», а оказаться в точке, где её просто нет. Через цепочку «Почему?» она находит корень и спрашивает: а что, если посредник между намерением и результатом вообще не нужен?

---

## Ценность

> Единственная ценность — оказаться в точке, где проблемы нет.

Человек не хочет «решить проблему». Он хочет, чтобы её просто не было. Не важно, как устроено решение. Важно, чтобы она была — а потом её не стало.

Но большинство систем, процессов и инструментов не убирают проблему. Они строят вокруг неё промежуточные слои — всё, что стоит между намерением человека и результатом: формы, статусы, согласования, маршруты, очереди, проверки, инструкции, графы состояний, workflow. Это костыли системы, которая не умеет самоисправляться. И человек вынужден быть посредником между своим намерением и результатом.

Почемучка спрашивает не «как быстрее пройти этот путь», а **«зачем вообще путь?»**

Через цепочку «Почему?» (3–7 шагов) она находит:

- **Корень** — что на самом деле порождает проблему.
- **Намерение** — как выглядит точка, где проблемы нет.
- **Принципы** — границы, при которых система сама оказывается в этой точке.

И в конце задаёт вопрос, который ломает привычную постановку: а что, если промежуточный слой между намерением и результатом вообще не нужен — потому что намерение можно описать напрямую? Что вы построите, когда посредник исчезнет?

---

## Как это работает

### 1. Онбординг
Если кейс расплывчатый — задаём 2–3 уточняющих вопроса и предлагаем 2–3 варианта тезиса.

### 2. Цепочка «Почему?»
От тезиса — через последовательные вопросы «Почему?» — до управляемого корня.

### 3. Корень
Один-два предложения: что на самом деле порождает наблюдаемое.

### 4. Линза
Переформулировка через три сдвига:

| Было | Стало | Для человека |
|------|-------|--------------|
| **Данные** | **Контекст** | Не «что есть», а **«что значит»** |
| **Проблемы** | **Намерение** | Не «что сломалось», а **«что должно быть»** |
| **Решения** | **Принципы** | Не «как делать», а **«на что ориентироваться»** |

> **Понимай ситуацию. Опиши желаемое. Доверься принципам.**

### 5. Побуждение к переходу
Открытый вопрос, который заставляет задуматься: а что, если промежуточный артефакт вообще не нужен?

## AI-перспектива

AI меняет не инструменты — он меняет саму модель взаимодействия. Всё, что требует человека как посредника между намерением и результатом, — это костыль системы, которая не умеет самоисправляться.

**Compiled Agents** — агент переводит намерение в workflow-код, человек ревьюит, в проде — детерминизм. Это мост.

**Intent Infrastructure** — человек описывает намерение + принципы напрямую, система исполняет. Это берег.

Почемучка не предписывает подход. Она заставляет спросить: а что, если посредник между намерением и результатом вообще перестаёт быть нужен?

## Установка

Есть три пути установки навыков в свой проект. (И конечно ты всегда можешь попросить установить навык своего агента не вникая в текст установки ниже.)

### Путь A: `npx skills add` (рекомендуемый)

Если установлен [agent-skills CLI](https://github.com/vercel-labs/agent-skills) от Vercel:

```bash
# Все навыки сразу
npx skills add https://github.com/kestgalax/pochemuchka

# Или по отдельности
npx skills add https://github.com/kestgalax/pochemuchka --skill "pochemuchka"
npx skills add https://github.com/kestgalax/pochemuchka --skill "pochemuchka-render"
```

*Примечание:* `npx skills add` требует установленного CLI `agent-skills`. Если он не установлен, используй Путь B или C.

### Путь B: `./setup.sh` или `setup.bat`

Bash-скрипт для ручной установки. Работает на macOS/Linux (Bash 3.2+). Детектирует IDE в проекте и копирует навыки в нужную директорию.

**macOS / Linux:**
```bash
./setup.sh                          # автоопределение IDE в текущем проекте
./setup.sh --all                    # во все найденные IDE
./setup.sh --target claude          # в конкретную IDE (opencode|claude|codex|cursor)
./setup.sh --global                 # в глобальную конфигурацию пользователя
./setup.sh --list                   # посмотреть, что будет установлено
./setup.sh --help                   # справка по всем флагам
```

**Windows:**
```cmd
setup.bat                          # автоопределение IDE в текущем проекте
setup.bat --all                    # во все найденные IDE
setup.bat --target claude          # в конкретную IDE (opencode|claude|codex|cursor)
setup.bat --global                 # в глобальную конфигурацию пользователя
setup.bat --list                   # посмотреть, что будет установлено
setup.bat --help                   # справка по всем флагам
```

### Путь C: Ручная установка

Скопировать папки из `skills/` в конфигурацию своего AI IDE:

```bash
# OpenCode
cp -r skills/pochemuchka .opencode/skills/
cp -r skills/pochemuchka-render .opencode/skills/

# Claude Code
cp -r skills/pochemuchka .claude/skills/
cp -r skills/pochemuchka-render .claude/skills/

# Или просто вставить содержимое `SKILL.md` в ChatGPT / Codex-чат
```

## Рабочая папка

Результаты анализа сохраняются в `pochemuchka-results/` (создаётся автоматически при первом сохранении):

```
pochemuchka-results/
├── 2026-05-29-service-desk.md
└── 2026-05-29-service-desk.html
```

- Markdown — полный текстовый разбор.
- HTML — автономная страница с тёмной/светлой темой, timeline, карточкой корня, таблицей линзы.

Папка добавлена в `.gitignore` и не попадает в коммиты.
